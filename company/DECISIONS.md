# 角落旅行社重大決策記錄

## 2026-03-17：景點資料庫多語言協作策略

### 背景
- 完成中文景點資料庫建置：2,558 個景點，34% 高品質描述
- 發現問題：純中文資料無法給地接社使用
- 需要多語言支援

### 決策：採用「協作式資料精煉」模式

#### 核心邏輯
1. **我們提供初稿**：中文 + AI 翻譯的英文/當地語言
2. **地接驗證優化**：第一家地接修正後存檔
3. **持續改進**：後續地接可在此基礎上再優化
4. **飛輪效應**：資料越準確 → 對地接越有價值 → 更多地接願意參與

#### 為什麼重要？

**短期價值**：
- 解決多語言問題
- 提升與地接協作效率
- 確保景點資料準確性

**中期價值**：
- 建立競爭壁壘（資料品質）
- 深化地接合作關係
- 降低每次做行程的查詢成本

**長期價值**：
- 資料庫成為業界標準
- 可能發展成獨立業務單元（資料授權/API服務）
- ERP 產品化的核心競爭力
- 建立地接社生態系統

#### 商業邏輯

這不只是「翻譯」，而是：

```
資料庫初稿 
  → 地接驗證（他們最懂當地）
  → 存檔成為標準版
  → 其他地接受益
  → 更多地接願意加入
  → 資料品質螺旋上升
  → 成為業界最準確的資料庫
  → 建立不可複製的競爭優勢
```

#### 類比案例
- Google Maps：用戶協作標註
- Wikipedia：全球志願者協作
- 角落資料庫：地接社協作驗證

#### 技術實作

**資料庫結構增加**：
```sql
- name_en (英文名稱)
- name_local (當地語言名稱)
- description_en (英文描述)
- description_local (當地語言描述)
- is_verified (是否經地接驗證)
- verified_by (驗證者)
- verified_at (驗證時間)
- notes_from_supplier (地接備註)
```

**協作流程**：
1. 我們準備初稿（中文 + AI 翻譯）
2. 發送給地接審核（Excel/線上表單）
3. 地接修正回傳
4. 更新資料庫並標記已驗證
5. 後續使用已驗證版本

#### 執行時程

**第一階段（立即）**：
- 增加多語言欄位
- 翻譯越南核心景點（69 個）英文初稿
- 建立協作模板

**第二階段（3 個月內）**：
- 與越南、日本第一批地接建立協作
- 完成 5 個主要國家核心景點驗證

**第三階段（6-12 個月）**：
- 資料品質達 60%+
- 開始有其他旅行社詢問
- 評估資料授權可能性

#### 風險與應對

**風險 1**：地接不願意花時間驗證
- **應對**：提供誘因（折扣/優先合作/積分）

**風險 2**：不同地接修改衝突
- **應對**：版本控制 + 以最新驗證為準

**風險 3**：品質控制
- **應對**：人工抽查 + 建立驗證標準

#### 預期成果

**6 個月後**：
- 5 個主要國家（台、日、韓、泰、越）核心景點完成多語言驗證
- 與 10+ 家地接建立協作關係
- 資料庫成為公司核心資產

**12 個月後**：
- 資料庫成為業界最準確的多語言旅遊景點資料庫
- 開始評估對外授權/API 服務可能性
- 成為 ERP 產品化的核心賣點
