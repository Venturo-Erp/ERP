## AI 的三個層次

### 層次 1：問答（ChatGPT / Gemini / 豆包）

**運作方式**：

```
你問 → AI 答 → 結束
```

**應用**：

- 寫文案
- 翻譯
- 簡單問答

**限制**：

- 每次都要重新問
- 無法自動執行

**[展示投影片：ChatGPT 對話截圖]**

---

### 層次 2：工作流自動化（n8n）

**運作方式**：

```
觸發條件 → 自動執行多個步驟 → 輸出結果
```

**具體範例**：

```
客戶填 Google 表單
    ↓ (自動觸發)
建立訂單到系統
    ↓ (自動執行)
發 LINE 通知業務
    ↓ (自動執行)
更新 Google Sheet 統計
    ↓ (自動執行)
發確認信給客戶
```

**關鍵差異**：

- ✅ 不用人工操作
- ✅ 一個動作觸發 5 個步驟
- ✅ 24 小時自動運作

**[展示投影片：n8n 工作流程圖]**

**金句**：

> 「這就是工作流自動化：

# Part 3：以前 vs 現在（3 分鐘）

## 場景：客戶想要 LINE@ 自動回覆訂單狀態

### 以前的做法（傳統開發）

**流程**：

```
第 1 週：找工程師
    - 上網徵才
    - 面試
    - 談價格

第 2 週：需求訪談
    - 工程師問：「你要什麼功能？」
    - 你說：「我也不知道...」
    - 來回溝通

第 3 週：設計 UI
    - 畫面長怎樣？
    - 按鈕放哪裡？
    - 顏色配色

第 4-7 週：寫程式
    - 串接 LINE API
    - 串接資料庫
    - 寫邏輯判斷
    - Debug 修 bug

第 8 週：測試上線
    - 測試各種情況
    - 修改 bug
    - 正式上線
```

**結果**：

- ⏰ **時間**：6-8 週（1.5-2 個月）
- 💰 **費用**：5-10 萬起跳
- 🤔 **風險**：做好了需求可能已經變了

**[展示投影片：傳統開發時間軸]**

---

### 現在的做法（AI 開發）

**流程**：

```
第 1 小時：告訴 AI 需求
    - 打開 AI 工具（如 Cursor / Windsurf）
    - 說：「我要 LINE@ 自動回覆訂單狀態」
    - AI 自動生成程式碼

第 2 小時：測試調整
    - 測試功能
    - 有問題跟 AI 說

第 3 小時：上線
    - 部署到伺服器
    - 開始運作
```

**結果**：

- ⏰ **時間**：2-3 小時（當天完成）
- 💰 **費用**：幾乎為零（或月費訂閱）

# 注意事項

## 時間控制

- 嚴格控制在 13 分鐘內
- 每句話不超過 20 字
- 適當停頓讓訊息沈澱

## 語氣

- 親和但專業
- 像同行在聊天
- 不說教、不恐嚇

## 禁用詞

- ❌ 革命性
- ❌ 顛覆
- ❌ 絕對
- ❌ 保證賺錢
- ❌ 比其他 ERP 更好（不直接比較）

## 展示原則

- 具體案例（不抽象）
- 有畫面（截圖、流程圖）
- 有數字（2 小時 vs 2 個月）
