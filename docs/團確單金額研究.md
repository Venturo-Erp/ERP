## ❓ 問題：「預計支出」應該用哪個欄位？

### 方案 A：用 `confirmed_cost`

```
預計支出 = confirmed_cost（確認後的價格）
```

**理由**：

- 確認後才出團，所以「預計」= 確認的價格
- 邏輯清楚

**問題**：

- 如果還沒確認，`confirmed_cost` = null
- 團確單顯示空白？

---

### 方案 B：用 `unit_price × quantity`

```
預計支出 = unit_price × quantity
```

**理由**：

- 永遠有值（只要有報價）
- 就算沒確認也能算

**問題**：

- 如果供應商報價不同（reply_cost ≠ unit_price）
- 預計支出可能不準

---

### 方案 C：優先級降級

```
預計支出 = confirmed_cost || reply_cost || unit_price × quantity
```

**理由**：

- 有確認用確認價
- 沒確認但有報價用報價
- 都沒有用估價

**優點**：

- 最準確
- 永遠有值

## 🔧 交通表的特殊邏輯

William 的範本：交通表**沒有**「商品單價」「數量」「小計」

| 日期  | 接駁地點       | 預計支出 | 實際支出 | 說明                 |
| ----- | -------------- | -------- | -------- | -------------------- |
| 03/21 | 機場>飯店>景點 | 35,000   | -        | 款項待行程結束後付款 |

**原因**：交通費用通常是**包車 / 整日報價**，不是「單價 × 數量」

### 實作方式：

```typescript
if (category === 'transport' || category === 'group-transport') {
  // 交通表：不顯示單價/數量/小計
  預計支出 = confirmed_cost || reply_cost || unit_price
} else {
  // 其他表：顯示單價/數量/小計
  商品單價 = unit_price
  數量 = quantity
  小計 = unit_price × quantity
  預計支出 = confirmed_cost || reply_cost || (unit_price × quantity)
}
```

## ✅ 結論：推薦方案

### 團確單金額邏輯

| 欄位         | 核心表來源                                                    | 計算邏輯                 |
| ------------ | ------------------------------------------------------------- | ------------------------ |
| **商品單價** | `unit_price`                                                  | 直接顯示（交通表不顯示） |
| **數量**     | `quantity`                                                    | 直接顯示（交通表不顯示） |
| **小計**     | `unit_price × quantity`                                       | 計算顯示（交通表不顯示） |
| **預計支出** | `confirmed_cost \|\| reply_cost \|\| (unit_price × quantity)` | 優先級降級               |
| **實際支出** | `actual_expense`                                              | 直接顯示（出團後回填）   |
| **說明**     | `expense_note \|\| confirmation_note \|\| quote_note`         | 優先級降級               |

### 財務彙總

```typescript
預計支出總金額 = SUM(預計支出)
實際支出總金額 = SUM(actual_expense)
```
