## 🎓 教訓：如何區分廢棄 vs 未完成

### 廢棄欄位的特徵

1. **有替代品** — 如 `op_staff_id` 被 `controller_id` 取代
2. **完全沒有業務邏輯** — 如 `last_login_at` 沒有更新機制
3. **使用次數 = 0** 或 極低（僅在 SELECT 但不使用值）

### 未完成功能欄位的特徵

1. **有明確業務邏輯** — 如付款日期、報價單號
2. **欄位名稱合理** — 一看就知道用途
3. **可能在 migration 中有註解** — 說明這是為了某功能準備的
4. **前端 UI 正在開發** — 只是功能還沒完成
