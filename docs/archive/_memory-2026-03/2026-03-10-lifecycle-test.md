# Venturo ERP 業務流程完整測試

**測試日期**: 2026-03-10  
**測試目標**: 測試請款單、收款單、出納單的完整生命週期和資料一致性  
**測試帳號**: SUCCESS / E001 / 12345678

---

## 測試策略

### 🎯 測試範圍

1. **請款單完整流程**
   - 新增 → 審核 → 變成出納單 → 付款 → 完成
   - 在各個狀態下，檢查顯示和統計是否正確
   - 刪除後，相關資料是否正確清除

2. **收款單完整流程**
   - 新增 → 確認收款 → 完成
   - 統計更新是否正確

3. **旅遊團整合顯示**
   - 請款單/收款單是否正確出現在團詳情
   - 財務總覽統計是否正確
   - 刪除後統計是否更新

4. **資料一致性**
   - 跨頁面資料同步
   - 刪除的連動效應
   - 統計數字的準確性

### Phase 2: 請款單完整流程測試

#### 2.1 新增請款單

- [ ] 在旅遊團中新增請款單
- [ ] 填寫完整資訊：
  - 選擇供應商
  - 填寫項目描述
  - 輸入金額（例如：NT$ 10,000）
- [ ] 提交請款單
- [ ] **驗證點 A**：請款單列表是否出現新記錄
- [ ] **驗證點 B**：團詳情頁面「請款統計」是否更新
- [ ] **驗證點 C**：請款單狀態是否為「待審核」

#### 2.2 請款單 → 出納單流程

- [ ] 審核請款單（改為「已審核」狀態）
- [ ] **驗證點 D**：是否自動產生出納單
- [ ] 進入出納單頁面
- [ ] **驗證點 E**：出納單是否包含正確的請款資訊
- [ ] **驗證點 F**：出納單金額是否正確（NT$ 10,000）

#### 2.3 出納單處理流程

- [ ] 標記出納單為「已付款」
- [ ] **驗證點 G**：請款單狀態是否更新為「已付款」
- [ ] **驗證點 H**：團詳情「已付款金額」是否更新
- [ ] **驗證點 I**：財務報表是否正確反映

#### 2.4 請款單刪除測試

- [ ] 新增第二筆測試請款單（NT$ 5,000）
- [ ] 記錄刪除前的統計數字
- [ ] 刪除第二筆請款單
- [ ] **驗證點 J**：請款單列表不再顯示已刪除記錄
- [ ] **驗證點 K**：團詳情統計數字是否正確減少（-5,000）
- [ ] **驗證點 L**：財務總覽統計是否更新
- [ ] **驗證點 M**：如果已產生出納單，出納單是否也被刪除
