## 一、訂單與客戶

### 客戶建立

- **有案子才有客戶** — 訂單是唯一源頭
- 過渡期保留「新增客戶」，未來關掉
- 客戶管理頁最終只保留「查看」和「編輯」

### 護照同步

- 訂單上傳護照 → 回寫客戶 + 檢查其他未出發訂單
- 衝突不自動覆蓋，跳提醒讓使用者選
- 已出發/已完成的訂單不動

### 訂單成員

- orders/order_members 是**硬刪除**（沒有 deleted_at）
- 一個訂單可有多人，每人有自己的護照
- 訂單刪除時團員可能變孤兒（已知問題）

## 二、出團

### 團的生命週期

```
提案(Proposal) → 開團(Tour) → 報價 → 收款 → 出發 → 結案
```

### 團與訂單的關係

- 一個團可對應多個訂單（多組客人參加同一團）
- 團沒有「業務」和「OP」欄位 — tours 表沒有這兩個欄位
- 顯示業務時是取第一個訂單的業務（不精確）
- current_participants 全為 0（歷史問題）

### 團的編輯（2026-03-05 新增）

- 團列表有編輯功能（團名等基本資訊）
- 附加費用欄位預設隱藏
- 分頁記憶（URL + localStorage 混合方案）

### 團確單

- 開團時自動建立（草稿）
- 供應商回覆自動更新
- 「產生團確單」= 「定案」按鈕，放在團確單頁面
- 定案 = 鎖定快照

## 三、報價

### 定價規則

- **成人與兒童價格不同**
- **6 歲以下不佔床**（佔床/不佔床影響價格）
- **嬰兒**（2 歲以下）通常免費或僅收稅金
- tours.price 存的是**每人成本**，不是售價
- 開團時售價資訊會丟失（已知問題）

### 報價單版本

- 每次修改產生新版本
- 舊版本保留，可回溯
- 報價單 ↔ 行程表雙向同步

## 四、財務

### 收款

- 支援：現金、轉帳、信用卡（LinkPay 台新）
- 分期付款：可設定多期
- 收款確認 ≠ 結案

### 請款

- 向供應商請款
- race condition 風險：收款和請款同時確認會互相覆蓋

### 代收轉付

- 旅遊業特殊制度：只對**服務費**課營業稅
- 代收轉付收據格式有法規要求
- 5% 管理費只顯示不寫 DB
- 12% 稅完全不存在於系統中

### 利潤計算

- 毛利率 8~15%，淨利率 3~8%
- 收款/請款/結案用三種不同口徑算 profit

### LinkPay（台新金流）

- MAC_KEY 必須有值（已加空值檢查）
- Webhook 已加 auth 驗證

## 五、供應商

### 需求單流程

- 需求總覽 → 勾選項目 → 發需求單 → 供應商回覆 → 回填團確單
- 一次可勾多個項目發給同一供應商
- 每項可加備註（報價/只訂位/不需要）
- 多幣別支援（JPY）

### 供應商角色

- 公司角色不固定，由需求單方向決定
- Local 報價模式：整包發需求，Local 回覆打包總價
- 按人數階梯報價
