## 🔴 上線前必須修

### A1. 支票頁整頁是騙（會計日常驚悚）
**檔案**：`src/app/(main)/accounting/checks/page.tsx`
點「標記已兌現」/「作廢」按鈕沒反應、重新整理還是原狀態、會計以為自己點錯。
**修法**：刪整頁或完整做。**不能留現況騙使用者**。

### A2. 科目「標為常用」幽靈欄位
**檔案**：`src/app/(main)/accounting/accounts/page.tsx` L131-135
點星號 → silent fail。是 Type 欺騙：TS 說有、SQL 說沒有。

### A3. 表單無 auto-save draft（業務員報價掉資料）
**痛點**：花 30 分鐘填完表單、網路中斷或 refresh → 全部消失。月 3-5 次、累積浪費。
**修法**（M、ROI 最高）：
```typescript
useEffect(() => {
  const timer = setTimeout(() => {
    localStorage.setItem(`draft_request_${tourId}`, JSON.stringify(formData))
  }, 2000)
  return () => clearTimeout(timer)
}, [formData])
```
**UX 成本**：每員工月掉 1-2 次（30 分 × 12 = 6 小時）× 10 員工 = 60 小時/月浪費。

### A4. 狀態值混用 '0'/'1' vs 'pending'（財務邏輯炸彈）
**現況**：`receipt.status` 用 `'0'/'1'`、`payment_request.status` 用 `'pending'/'confirmed'`。
**後果**：報表每月錯 1-2 次、月底對帳卡 2 小時。

### A5. 業務術語混亂（「確認」vs「成交」vs「已審核」）
**痛點**：新員工看界面分不清「待確認」是業務確認還是客戶確認。
**修法**：
1. 製作「業務術語 ↔ 系統術語」對照表（放 BUSINESS_MAP.md）
2. UI label 加 tooltip
3. 員工 onboarding 讀對照表

**UX 成本**：新員工 3 個月內每週問一次「這是什麼意思」× 3-5 人。

## 🟠 會被抱怨（上線 1-2 週開 ticket）

### B1. Loading state 不一致
部分頁用 `<ModuleLoading fullscreen />`（太 aggressive）、部分用 `{isLoading ? "載入中..." : <Table />}`（太素）、部分完全沒。按鈕提交無 disabled、使用者點 3 次。

**修法**：統一 skeleton pattern + 按鈕 `disabled={isSaving}`。

### B2. Empty state 無引導
新租戶看「團況」空白、沒有「新增」按鈕、沒有指引。

**修法**：
```tsx
<EmptyState
  icon={<FileText />}
  title="還沒有報價單"
  description="開始建立您的第一份報價單。"
  action={<Button onClick={...}>新增報價單</Button>}
/>
```

### B3. Error message 不友善
「發生未知錯誤」、不知道為什麼失敗。

**修法**：分類 error、給具體原因 + recovery hint。

### B4. 表單驗證時機不對（填完才知道錯）
目前只在 submit 時驗證、無 blur 驗證、表單太長看不到上面錯誤。

### B5. customers vs traveler_profiles 資訊重複
加訂單成員要重填資料。已列 Post-Launch。
