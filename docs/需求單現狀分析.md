# 需求單現狀分析

## 📋 tour_requests 表結構（現有欄位）

### 基本資訊
| 欄位 | 說明 | 現況 |
|------|------|------|
| `id` | 需求單 ID | ✅ 有 |
| `code` | 需求單編號 | ✅ 有 |
| `tour_id` | 所屬團號 | ✅ 有 |
| `request_type` | 類型（accommodation/meal/transport/activity/other） | ✅ 有 |
| `items` | 需求項目（JSON） | ✅ 有 |
| `status` | 狀態（draft/sent/replied/confirmed/cancelled） | ✅ 有 |

### 供應商資訊
| 欄位 | 說明 | 現況 |
|------|------|------|
| `supplier_name` | 供應商名稱 | ✅ 有 |
| `supplier_id` | 供應商 ID | ✅ 有 |
| `supplier_contact` | 供應商聯絡方式 | ✅ 有 |
| `supplier_response` | 供應商回覆（JSON） | ✅ 有 |

### 流程追蹤
| 欄位 | 說明 | 現況 |
|------|------|------|
| `created_at` | 建立時間 | ✅ 有 |
| `sent_at` | 發送時間 | ✅ 有 |
| `sent_via` | 發送方式（LINE/Email/...） | ✅ 有 |
| `sent_to` | 發送給誰（群組 ID/Email） | ✅ 有 |
| `replied_at` | 回覆時間 | ✅ 有 |
| `confirmed_at` | 確認時間 | ✅ 有 |

### 來源追蹤
| 欄位 | 說明 | 現況 |
|------|------|------|
| `source_type` | 來源類型（itinerary_item） | ✅ 有 |
| `source_id` | 來源 ID（核心表 item.id） | ✅ 有 |

### 其他
| 欄位 | 說明 | 現況 |
|------|------|------|
| `note` | 備註 | ✅ 有 |

```json
  "vehicleType": "43人座遊覽車",

      "date": "2026-03-21",

```

```json

  {
    "meal_time": "午餐",
```
