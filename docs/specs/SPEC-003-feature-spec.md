## 👤 使用者故事

### Story 1：快速新增任務

**身為 William**  
我想要快速新增任務

5. 自動分配負責人（可修改）
6. 點擊「建立」

## 🎨 核心功能

### 1. 任務看板（Kanban）

**4 欄佈局**：
- **P0**：緊急重要（紅色，最多 5 個）
- **P1**：重要（橙色）
- **P2**：一般（藍色）
- **完成**：已完成（綠色，只能放不能拖出）

**任務卡片元素**：
- 優先級標籤
- 負責人頭像（最多 3 個）
- 任務標題
- 期限（過期紅色警告）
- 標籤（最多 3 個）
- 預計時間
- 進度條（根據子任務自動計算）
- 子任務列表（最多顯示 3 個）
- 快速操作：編輯、完成、刪除、指派

---

### 2. 拖曳功能

**支援操作**：
- 拖曳任務到不同優先級欄位
- 拖曳任務到「完成」欄位
- 拖曳子任務調整順序（詳細頁）

**限制**：
- P0 欄位最多 5 個任務（超過警告）
- 完成欄位只能放，不能拖出
- 已完成任務不能拖回待辦

---

### 3. 工作流範本

**3 個預設範本**：
1. **數字人開發**（5-7 天）
2. **活動企劃**（3-5 天）
3. **功能開發**（3-10 天）

**範本包含**：
- 預設子任務
- 預設負責人
- 預計時間
- 檢查點（需審批）

**使用方式**：
1. 新增任務時選擇範本
2. 系統自動產生子任務
3. 可修改負責人、時間

---

### 4. 任務詳情（Side Panel）

**點擊任務卡片展開**：
- 完整子任務列表（可編輯、勾選）
- 評論區（團隊溝通）
- 附件上傳（最多 10MB）
- 活動紀錄（誰做了什麼）
- 編輯按鈕

---

### 5. 快速操作

**不需打開詳情頁**：
- 雙擊標題：直接編輯
- 雙擊日期：彈出日期選擇器
- 點擊進度條：快速編輯子任務
- 點擊負責人：重新指派

## 🗂️ 資料結構

### 資料表：tasks

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | uuid | 主鍵 |
| title | text | 任務標題 |
| description | text | 任務描述 |
| priority | text | P0/P1/P2 |
| status | text | todo/in_progress/done |
| assignees | jsonb | 負責人陣列 |
| due_date | date | 期限 |
| estimated_days | int | 預計天數 |
| tags | jsonb | 標籤陣列 |
| workflow_template | text | 工作流範本代號 |
| created_at | timestamp | 建立時間 |
| updated_at | timestamp | 更新時間 |
| created_by | uuid | 建立者 |

---

### 資料表：task_steps（子任務）

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | uuid | 主鍵 |
| task_id | uuid | 父任務 ID |
| title | text | 子任務標題 |
| status | text | todo/in_progress/done |
| assignee | uuid | 負責人 |
| order | int | 排序 |
| created_at | timestamp | 建立時間 |
| completed_at | timestamp | 完成時間 |

---

### 資料表：workflow_templates（工作流範本）

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | uuid | 主鍵 |
| code | text | 範本代號 |
| name | text | 範本名稱 |
| duration_days | text | 預計天數範圍 |
| steps | jsonb | 步驟定義（YAML 轉 JSON） |
| created_at | timestamp | 建立時間 |

## 🚨 邊界案例

### 1. 中途加人

**情境**：任務進行中，需要加入新負責人

**處理**：
- 點擊任務 → 編輯 → 新增負責人
- 系統發送通知給新負責人

---

### 2. 任務暫停

**情境**：任務需要暫停（等待外部資源）

**處理**：
- 新增「暫停」狀態（灰色）
- 暫停任務不計入進度

---

### 3. 任務取消

**情境**：任務不做了

**處理**：
- 點擊「刪除」→ 確認 → 軟刪除（不真正刪除，只是隱藏）
- 可在「已刪除」篩選器中找回

---

### 4. 子任務超過 10 個

**情境**：複雜任務有很多子任務

**處理**：
- 卡片只顯示前 3 個
- 顯示「+7 個子任務」
- 點擊展開完整列表

---

### 5. 多人同時編輯

**情境**：兩個人同時編輯同一個任務

**處理**：
- Supabase Realtime 即時同步
- 顯示「XXX 正在編輯」提示
- 後寫入的覆蓋（簡單處理）
