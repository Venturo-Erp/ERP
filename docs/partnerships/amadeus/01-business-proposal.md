## 1. 提案背景

Venturo 面向台灣旅遊業推出新一代 SaaS ERP 平台，
內建航班查詢、訂位、出票、異動通知等全流程模組，
**需要 Amadeus 作為底層 GDS 合作夥伴**，
共同開拓台灣中小型旅行社市場。

本提案的核心商業邏輯：

> **一次性完整 Scope 合約**：Venturo 承諾一次簽下涵蓋未來 24 個月所有功能的 API 授權，
> Amadeus 只收一次整合費，避免 Venturo 分階段追加帶來的管理成本與議約次數。

## 3. 市場機會分析

### 3.1 台灣旅行社概況

| 類別              | 家數        | IATA 狀態  | GDS 能力   |
| ----------------- | ----------- | ---------- | ---------- |
| 大型綜合          | ~50         | 多數有     | 自有團隊   |
| 中型              | ~300        | 部分有     | 有限       |
| **中小型 / 新進** | **~2,500+** | **多數無** | **幾乎無** |

### 3.2 中小型旅行社的三重痛點

1. **無 IATA 資格** → 查位子要打給大票代、等回覆、效率低
2. **不會 GDS 指令** → 業務訓練曲線高，新人上手慢
3. **流程分散** → GDS 終端機 + LINE + Excel + 手寫單據，易出錯

### 3.3 現有解決方案的不足

- **純 OTA 工具**（Skyscanner 等）：只查不訂，無 ERP 整合
- **大型 GDS 終端機**：門檻高、費用貴、不適合小旅行社
- **票代服務**：手續費高、等候時間長、無自助能力

**市場缺口：一個給「無 IATA、不會 GDS 指令」的旅行社使用的平台。**

## 4. Venturo 的產品解決方案

### 4.1 產品架構

```
┌─────────────────────────────────────────┐
│     Venturo ERP（既有核心）               │
│  ┌───────────────────────────────────┐  │
│  │ 客戶 / 報價 / 訂單 / 會計 / 代收轉付 │  │
│  └───────────────────────────────────┘  │
│                  ▲                       │
│  ┌───────────────┴───────────────────┐  │
│  │  Amadeus 整合模組（本提案新增）      │  │
│  │                                    │  │
│  │  ┌─────────────┐  ┌────────────┐  │  │
│  │  │ 簡單模式     │  │ 專業模式    │  │  │
│  │  │ (C 端 UX)   │  │ (GDS 指令)  │  │  │
│  │  └─────────────┘  └────────────┘  │  │
│  │                                    │  │
│  │  查詢 / 訂位 / 出票 / 異動通知       │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
                  │
                  ▼
          Amadeus GDS APIs
```

### 4.2 雙模式介面設計（核心差異化）

| 模式                 | 適合對象             | 介面特性                          |
| -------------------- | -------------------- | --------------------------------- |
| **簡單模式**（預設） | 小旅行社、新手業務   | Expedia / Skyscanner 式消費者介面 |
| **專業模式**         | 資深業務、大量操作者 | 類 GDS 指令 + 快捷鍵              |

**使用者可即時切換**。新人入門用簡單模式，熟練後切專業模式加速。

### 4.3 殺手功能：航班異動自動通知

- 整合 Amadeus Queue Management（QJ / TJ / EJ）與 Schedule Recovery API
- 綁定 PNR → 自動監控 → SSIM / TK / UN 訊息即時推播
- 通知業務 + 旅客 + 自動草擬改票方案
- **小旅行社不再需要 24 小時人工巡 Queue**

### 4.4 ERP 閉環整合

訂位完成 → PNR 自動進 Venturo 訂單模組 → 自動關聯客戶檔、會計、代收轉付 → 對帳一鍵完成。

## 5. 分階段落地計畫（合約仍為一次性）

**重要：本節為 Venturo 內部開發節奏，Amadeus 只需一次性授權完整 Scope。**

### 階段 1（0-6 個月）：查詢 + 異動通知

- Flight Search / Availability / Fare Quote
- Queue Management + Schedule Recovery
- 50 家旅行社試營運

### 階段 2（6-12 個月）：訂位整合

- PNR Create / Modify / Cancel
- Seat Selection / SSR
- 100-200 家旅行社全面上線

### 階段 3（12-24 個月）：出票 + 進階

- E-Ticket Issuance / Void / Refund / EMD
- BSP 整合（若 Venturo 取得票務授權）
- 300+ 家旅行社規模

## 7. 合作訴求

### 7.1 商業條款

1. **一次性完整 Scope API 授權合約**
2. **多租戶 / SaaS 白標明確允許**：Venturo 可讓旗下 SaaS 客戶（旅行社）透過 Venturo 使用 Amadeus API
3. **針對台灣中小旅行社的合理費率**
4. **量級遞減折扣**：月訂位量達某門檻後享費率優惠
5. **合約期間至少 3 年**，含續約優先權

### 7.2 技術支援

1. **專屬技術窗口**（Account Manager + 技術工程師）
2. **Sandbox / Test Environment 永久存取**
3. **SLA 明確**：API 可用性 99.5%+、故障通報時效
4. **API 版本升級保障**：新版本推出後，舊版至少再支援 18 個月

### 7.3 市場面（可選）

1. **聯合行銷**：共同參加台灣旅遊業展會、研討會
2. **案例背書**：Amadeus 可引用 Venturo 作為 APAC SaaS 成功案例
3. **繁中在地化**：Amadeus 文件、錯誤訊息繁中化由 Venturo 協助

## 8. 給 Amadeus 的獨特價值

### 8.1 打入原本難以服務的長尾市場

Amadeus 直接服務的多為中大型旅行社。台灣 2,500+ 家小旅行社因規模小，不適合直接合約。**Venturo 作為中介層**，用 SaaS 模式聚合這個市場，讓 Amadeus 一次觸達所有人。

### 8.2 繁中在地化由 Venturo 負擔

介面、錯誤訊息、客服支援繁中化全由 Venturo 承擔。**Amadeus 省下在地化團隊成本**。

### 8.3 生態擴大

Venturo 提供 ERP 層，讓 Amadeus 的 API 不只是訂票工具，而是**整合進旅行社每日作業的入口**。Amadeus 黏著度大幅提升。

### 8.4 APAC 市場示範

若台灣市場成功，可複製到東南亞、日韓（同樣多的中小型旅行社）。
