# 2026-03-28 收款模組重構

## 收款單編號規則（已確認）
- **單筆收款**：`R01`
- **多筆同時收**：`R01-A`, `R01-B`, `R01-C`...（全部帶後綴）
- **下一次收款**：`R02`（或 `R02-A`, `R02-B`...）
- `batch_id` 用第一筆的 ID，同批次共用

## 收款確認流程（已實作）
- 統一入口：編輯對話框
- **一般員工**：儲存
- **會計/管理員**：儲存 + 異常 + 確認
- 狀態：0=待確認, 1=已確認, 2=異常

## 修復的 Bug

### payment_method DB constraint
- DB 只允許：`cash`, `transfer`, `card`, `check`, `linkpay`
- 之前程式碼傳中文（`匯款`、`現金`）→ 已改成英文

### receipts list select 漏欄位
- `notes` 和 `batch_id` 沒在 select 裡 → 已加入
- 導致編輯時讀不到備註

### 建立收款時欄位沒存
- 之前版本第一筆寫死 `receipt_account: null` → 已修正為 `item.receipt_account`
- 舊資料需手動補填

### 收款確認後沒重算團財務
- 確認/異常後要呼叫 `recalculateReceiptStats`
- 更新 `tours.total_revenue` 和 `orders.paid_amount`
