📐 統一後的參考架構

🎯 三層架構（這次統一採用的標準）

  • 第一層：通用訊息 → src/constants/messages.ts（alert、toast、錯誤訊息）
  • 第二層：中央 Labels → src/constants/labels.ts（跨模組共用，如 accounting、quotes）
  • 第三層：模組 Labels → src/app/(main)/xxx/constants/labels.ts（頁面專屬）
  • Status 配置 → src/lib/status-config.ts（最佳實踐，color/label/icon 集中管理）

✅ 正確寫法（統一後的標準）

  • Alert：改走 import { COMMON_MESSAGES } from '@/constants/messages'
  • 頁面標題：改走 import { ACCOUNTING_PAGE_LABELS } from '@/constants/labels'
  • 模組 placeholder：改走 import { LABELS } from '../constants/labels'
  • Inline style：改走 Tailwind class（靜態值）或保留（動態值）
  • Tooltip / title：改走對應的 labels 常數

❌ 禁止寫法（清理目標）

  • alert('載入失敗')、alert('請選擇日期') 等裸字串
  • title="傳票管理"、title="資產負債表" 等硬編碼頁面標題
  • placeholder="輸入目前密碼" 等硬編碼 placeholder
  • style={{ background: 'linear-gradient(...)', boxShadow: '...' }} 大面積 inline style
  • <style>{`...`}</style> JSX 內嵌 CSS

🧪 驗證建議

如果你擔心我做錯，可以這樣快速驗證：

  • 打開這份 HTML，對照你心中的規範看「正確寫法」那一區是否一致
  • 跑 cd venturo-erp && npx tsc --noEmit 確認沒有編譯錯誤
  • 打開幾個改過的頁面（accounting、settings/personal、payslip）確認 UI 沒有跑版
  • 如果發現某個 label 應該是 A 但我寫成 B，直接改那個模組的 constants/labels.ts 即可，所有引用會自動更新
