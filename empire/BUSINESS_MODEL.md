## 📜 一句話

> **旅行社 = 資訊差的搬運工。**
>
> 客人不知道供應商的成本價，
> 供應商不知道客人的支付價，
> 旅行社站在中間，靠「資訊整合能力」吃飯。
>
> Venturo 讓這個搬運過程自動化、透明化、可追蹤。

## 🏗️ 一團的生命週期（錢的視角）

### 第一章：估價

```
勇者（業務）開團
  │
  ▼
🌳 世界樹種下 → 行程骨架（Day1~DayN）
  │
  ▼
填「成本」（估價階段）
  ├── 機票 $12,000（已知，航空公司報價）
  ├── 飯店 $3,000/晚 × 5 晚（估的）
  ├── 餐廳 $800/餐 × 10 餐（估的）
  ├── 景點 $500/人 × 3 個（估的）
  ├── 交通 $15,000/天 × 7 天（估的）
  ├── 導遊 $3,000/天 × 7 天
  └── 雜支 $5,000（保險/簽證/小費）

成本合計（估）: $150,000/人

  │
  ▼
加利潤 → 定「團費」
  團費 = 成本 + 利潤 + 風險緩衝
  $150,000 + $20,000 + $5,000 = $175,000/人

  │
  ▼
報價給客人
```

**關鍵**：此時成本全是「估的」，團費已定。
**風險**：估錯了，利潤就變了。估太低 = 賠錢。

---

### 第二章：採購

```
行程確定 → 開始發委託
  │
  ▼
勾選項目 → 發給供應商
  │
  ├── 直接供應商（國內操作）
  │   飯店 → 回覆 $2,800/晚（比估的便宜！）
  │   餐廳 → 回覆 $850/餐（比估的貴一點）
  │   車行 → 回覆 $14,000/天
  │
  └── 附屬國 Local（國外操作）
      打包委託（飯店+餐+活動+交通）
      → Local 回覆 $XXX（含 Local 利潤）

  │
  ▼
回填世界樹 → 成本修正
  飯店 $3,000 → $2,800（多賺 $200）✅
  餐廳 $800 → $850（少賺 $50）⚠️
  世界樹提醒：「飯店多賺 $200，餐廳少賺 $50」
```

**創世神聖諭**：
> 「成本就一個數字，從估價一路修正到最終。」

---

### 第三章：確認

```
供應商回覆 → 確認預訂
  │
  ▼
所有項目 confirmed → 生成「團確單」
  │
  ▼
團確單 = 領隊的作戰地圖
  Day1: 飯店 A（確認號 XXX）、餐廳 B（12:00 訂位）
  Day2: 景點 C（09:00 入場）、飯店 A（續住）
  ...

  │
  ▼
交給領隊 → 出發！
```

---

### 第四章：執行 & 結算

```
出團期間
  │
  ├── 領隊收到實際收據
  │   飯店實際 $2,750（比確認價又便宜一點）
  │   臨時加了一餐 $600
  │
  ▼
回團 → 結帳
  │
  ▼
回填世界樹 → 最終成本
  飯店 $2,800 → $2,750（又多賺 $50）
  加了一餐 $600（新增項目）
  
  │
  ▼
💰 損益計算
  團費收入：$175,000 × 20 人 = $3,500,000
  實際成本：$145,000 × 20 人 = $2,900,000
  ─────────────────────────────
  毛利：$600,000
  人均毛利：$30,000
  毛利率：17.1%
```

## 📊 帝國收入模型

### 現在（第一紀元）：自用

```
角落旅行社 用 Venturo ERP
  └── 降低 OP 工時 → 降低人力成本
  └── 減少估價錯誤 → 保護利潤
  └── 自動追蹤供應商 → 減少漏單
  
  收入 = $0（自建自用）
  價值 = 省下的人力 + 避免的損失
```

### 未來（第五紀元）：平台

```
Venturo 平台
  │
  ├── 🏛️ SaaS 月費
  │   旅行社 A → $X/月
  │   旅行社 B → $X/月
  │   Local C  → $Y/月（較低）
  │
  ├── 🔗 交易抽成（可選）
  │   旅行社 → Local 的委託，平台抽 X%
  │   （因為平台媒合了供需）
  │
  ├── 📊 數據服務（進階）

  │
  └── 🎓 帝國學院（培訓）

      供應商系統對接培訓
```

## ⚖️ 商業模式的前後因果

### 為什麼需要核心表（世界樹）？

```
沒有核心表：
  業務估價 → 手寫 Excel
  助理訂房 → 另一個 Excel
  領隊帶團 → 印出來的紙
  會計結帳 → 又一個 Excel
  → 四份不同的數字，對不起來
  → 賠錢都不知道賠在哪

有核心表：
  估價 → 寫在世界樹
  訂房 → 修正世界樹
  帶團 → 讀世界樹
  結帳 → 比對世界樹
  → 一份數字，從頭到尾可追蹤
  → 每個環節的差異一目了然
```

### 為什麼需要附屬國系統？

```
沒有附屬國系統：
  發 Email/傳真給 Local
  → Local 用自己的系統報價
  → 格式不統一，要人工轉錄
  → 改行程要再發一次
  → 確認狀態靠 LINE 追蹤
  → 帳目對不起來

有附屬國系統：
  發委託 → Local 在同一套系統收到
  → 格式統一，直接填價
  → 改行程 = diff，雙方看得到
  → 狀態即時追蹤
  → 帳目自動對齊
```

### 為什麼需要記憶魔法？

```
沒有記憶魔法：
  新 OP 上手要 6 個月
  換人 = 重新學
  教訓不會累積
  同樣的錯反覆犯

有記憶魔法：
  帝國知識 = 共享資產
  AI 公民帶著記憶工作
  教訓自動傳承
  「去年越南團的飯店太差」→ 今年自動提醒
```

## 🎯 商業護城河

```
第一層：工具效率
  「用 Venturo 比 Excel 快 10 倍」
  → 容易被抄

第二層：流程標準化
  「Venturo 定義了旅遊業的標準表格」
  → 中等護城河

第三層：網絡效應 ← 真正的護城河
  「旅行社和 Local 都在 Venturo 上」
  → 旅行社越多 → Local 越想加入
  → Local 越多 → 旅行社越想用
  → 像 Airbnb：房東和房客互相吸引

第四層：數據資產
  「Venturo 知道每條路線的真實成本」
  → 行業定價基準
  → 供應商評分
  → 市場趨勢預測
  → 這是別人沒有的
```
