## 🎯 核心業務原則（5條鐵律）

```
1. 有案子才有客戶 → 訂單是唯一源頭
2. 核心表是唯一真相 → tour_itinerary_items
3. 代收轉付制度 → 毛利 8-15%
4. 多租戶隔離 → workspace_id + RLS
5. 個人分攤報價 → 不是總價
```

## 💰 財務規則

### 毛利率標準

```
毛利率：8-15%
淨利率：3-8%

計算口徑（3種）：
1. 收款階段 → 預估毛利
2. 請款階段 → 實際支出毛利
3. 結案階段 → 最終毛利
```

### 代收轉付制度

```
流程：
  客戶付款 → 公司帳戶（代收）
    ↓
  支付供應商 → 公司帳戶（代付）
    ↓
  毛利 = 代收 - 代付

特點：
  - 不預付款給供應商
  - 收到客戶款才付
  - 現金流安全
```

### 收款方式

```
支援：
1. 現金
2. 轉帳
3. 信用卡（LinkPay 台新金流）

分期付款：
  - 可設定分期
  - 記錄每期狀態
```

### 匯率處理

```
報價階段：
  - 用保守匯率（JPY 0.21~0.22）
  - 業務自己抓 buffer
  - 轉成 TWD 報價

實際支出：
  - 每筆獨立記錄原幣 + 匯率 + TWD
  - 匯率差價歸公司
  - 外幣庫存不管（未來會計系統）
```

## 💵 報價規則

### 個人分攤（不是總價）

```
❌ 錯誤：一蘭拉麵 $30,000（30人總共）
✅ 正確：一蘭拉麵 $1,000/人

原因：
  - 業務要算「每個人付多少」
  - 直接填個人分攤更清楚
  - 方便比較不同方案
```

### 價格分類

```
成人/兒童/嬰兒：
  - 成人價（adult_price）
  - 兒童價（child_price）
  - 嬰兒價（infant_price）

佔床/不佔床：
  - 影響住宿價格
  - 影響餐食（早餐）
```

### Local 報價模式

```
階梯報價：
  10人 → $5,000/人
  20人 → $4,000/人
  30人 → $3,333/人

規則：
  - 人數越多，單價越低
  - 自動判斷適用階梯
  - 多列顯示（每個階梯一列）
```

## 📋 需求單規則

### 自動帶入總人數

```
來源：
  SELECT SUM(adult + child_with_bed + child_no_bed + infant)
  FROM orders
  WHERE tour_id = ? AND status IN ('confirmed', 'paid')

用途：
  - 需求單自動帶入
  - 供應商知道總人數
  - 助理根據狀況決定桌數/房間數
```

### 桌數/房間數不自動計算

```
❌ 不自動：30人 ÷ 10人/桌 = 3桌

✅ 空白讓助理填：
  - 餐廳可能只有 8人桌
  - 可能有包廂
  - 可能要分男女桌
  - 助理最清楚狀況
```

## 🎯 快速診斷

### 毛利率不對

```
檢查：
1. 報價是否個人分攤
2. 成本是否完整
3. 是否有漏項
4. Local 報價是否正確
```

### 需求單人數錯誤

```
檢查：
1. 訂單狀態（只計 confirmed + paid）
2. 團員數量是否正確
3. 是否有草稿訂單
```

### 核心表資料不同步

```
檢查：
1. 是否直接改其他表（錯誤）
2. 是否透過核心表寫入（正確）
3. 同步機制是否執行
```
