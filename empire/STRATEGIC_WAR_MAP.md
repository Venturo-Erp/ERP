### 📜 第二戰役：武裝（✅ 已勝）

**目標**：讓助理能發委託

```
戰果：
  ✅ 委託書列印（住宿/餐食/活動/交通）
  ✅ 勾選項目打包發給供應商
  ✅ 年齡分類自動計算
  ✅ 住宿變更警報

意義：
  助理按幾個按鈕就能產出專業委託書
```

### ⚔️ 第三戰役：追蹤（🔨 進行中）

**目標**：讓委託有生命週期，能追蹤到底

```
進度：
  ✅ 委託存入 DB（tour_requests）
  ✅ 已發委託列表 + 狀態 Badge
  ✅ 委託展開詳情 + 狀態流轉按鈕
  ✅ 回填世界樹（確認時 quoted_cost → unit_cost）
  📋 團確單生成（confirmed 彙整給領隊）
  📋 結團 tab 串接（估價 vs 實際損益）

意義：
  從「能發」到「能追」
  助理知道哪張委託還沒回
  業務知道成本有沒有超
```

### 🏘️ 第五戰役：結盟（📋 40% 基礎已在）

**目標**：讓 Local 加入帝國，用同一套系統

```
已有的基礎：
  ✅ 租戶系統（workspace + RLS）
  ✅ 租戶建立 API + 初始化腳本
  ✅ 供應商財務頁面（空殼）

需要建的：
  📋 Local 收到委託 → 自動建案
  📋 Fork 世界樹枝條（複製行程到 Local workspace）
  📋 Local 用同一套行程編輯器操作
  📋 Local 逐項報價回覆
  📋 Local 請款 → 旅行社收到
  📋 行程 diff 追蹤（改了什麼）

意義：
  這是帝國擴張的第一步
  Local 不用再 LINE 接單、Word 報價
  旅行社和 Local 在同一棵世界樹下工作
  → 「統一協議」比「統一系統」更重要

戰略價值：
  每收編一個 Local = 網絡效應 +1
  Local 用了 Venturo → 它的其他旅行社客戶也會被吸引
```

### 🌅 第六戰役：統一（💭 願景）

**目標**：讓整個旅遊業都跑在 Venturo 上

```
三條攻擊路線：

路線 A：供應商自助
  飯店/車行/餐廳 → 線上收委託、線上回覆
  不用再等傳真、不用再手動填
  → 供應商越多，旅行社越方便
  → 旅行社越多，供應商越想加入

路線 B：其他旅行社加入
  SaaS 月費制
  「你看角落旅行社用得多順」
  → 標準化的表格和流程
  → 帝國的知識（景點庫、飯店庫）共享

路線 C：B2C 客人端
  Online 已有 18 個頁面骨架
  行程瀏覽 → 報名 → 付款 → 出發準備
  → 客人的錢直接進系統，不用手動對帳

最終形態：
  客人（Online）→ 旅行社（ERP）→ Local（同套系統）→ 供應商（自助回覆）
  全鏈條在 Venturo 上，每一筆錢、每一項服務都可追蹤
```

## 🎯 侵略路線

```
Phase 1：鞏固（現在~三紀完成）
  → 角落旅行社自己用得順
  → 助理能完整操作：開團→報價→發委託→追蹤→結團

Phase 2：內治（四紀）
  → 財務自動化串接
  → 每團損益一目了然
  → 人事管理上線

Phase 3：第一批附屬國（五紀）
  → 角落旅行社常合作的 Local 先上
  → 「你不用再 LINE 報價了，直接在系統填」
  → 驗證跨租戶協作

Phase 4：開放加盟（六紀）
  → 其他旅行社看到效果，付費加入
  → 供應商被動加入（因為旅行社都用 Venturo 發委託）
  → B2C 上線，客人直接報名

Phase 5：行業標準（六紀之後）
  → Venturo 的表格格式 = 行業標準
  → Venturo 的數據 = 行業基準
  → 「旅遊業的 Stripe」
```
