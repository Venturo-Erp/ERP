## 🏛️ 帝國六大機構

### 1️⃣ 🌳 核心聖殿 — 世界樹守護者

**對應系統**：核心表 `tour_itinerary_items`  
**守護者**：帝國建築師 Matthew

```
職責：
  - 維護世界樹的健康（資料完整性）
  - 確保所有人看到的是同一棵樹（SSOT）
  - 防止枝條腐敗（資料異常偵測）

世界樹的五面：
  🍃 東面 → 行程表（規劃視角）     → /tours/[id] Tab:行程
  🍃 南面 → 報價單（成本視角）     → /tours/[id] Tab:報價
  🍃 西面 → 需求單（採購視角）     → /tours/[id] Tab:需求
  🍃 北面 → 確認單（交接視角）     → /tours/[id] Tab:追蹤
  🍂 落葉 → 結帳單（結算視角）     → /tours/[id] Tab:已結團

世界樹的枝幹欄位：
  ├── 身份：id, tour_id, day_number, sort_order
  ├── 內容：category, sub_category, title, description
  ├── 金幣：unit_price, quantity, total_cost
  │         adult_price, child_price, infant_price
  ├── 任務：quote_status, request_status
  │         confirmation_status, leader_status
  ├── 回覆：quoted_cost, confirmed_cost, actual_expense
  ├── 時間：request_sent_at, request_reply_at
  │         confirmed_at, expense_at
  └── 連結：resource_id, resource_type, accommodation_id
```

### 2️⃣ ⚔️ 任務大廳 — 委託佈告欄

**對應系統**：需求單系統  
**管理者**：盜賊（助理）

```
位置：/tours/[id] Tab:需求

任務發布流程：
  ┌─────────────────────────────────────────┐
  │  佈告欄（需求頁）                         │
  │                                          │
  │  ☑ Day1 午餐 — 某餐廳                    │
  │  ☑ Day2 晚餐 — 某餐廳                    │
  │  ☐ Day2 自理餐                           │
  │  ☑ Day3 活動 — 和服體驗                   │
  │  ☑ 住宿 — 指定飯店（填房型×間數）         │
  │  ☑ 交通 — 遊覽車（勾選天數）              │
  │                                          │
  │  [ 已選 4 項 → 發給供應商 ]               │
  └─────────────────────────────────────────┘

委託書類型：
  🏨 住宿委託：填房型間數 → RoomRequirementDialog
  🍽️ 餐食委託：直接列印（助理看品名就知道）
  🎯 活動委託：直接列印
  🚌 交通委託：勾選天數 → TransportRequirementDialog
  📦 打包委託：勾選多項 → AssignSupplierDialog → 發給 Local

委託書共用資訊：
  我方：公司 / 團號 / 團名 / 出發日 / 人數 / 年齡分類
  對方：供應商名 / 窗口 / 電話 / Email / 地址
  明細：日期 / 類別 / 項目 / 預算 / 回覆金額 / 確認欄
  簽章：供應商簽章 + 日期

任務狀態機：
  draft（業務填完）
    → sent（助理發出）
    → replied（供應商回覆）
    → confirmed（確認預訂）
    → closed（結案）
```

### 5️⃣ 📜 帝國族譜 — 租戶血脈

**對應系統**：租戶系統（Multi-tenant）

```
族譜結構：

  🏰 帝國（Venturo 平台）
  │
  ├── 🏛️ 旅行社 A（workspace_id: aaa）← 我們（角落旅行社）
  │   ├── 員工 5 人
  │   ├── 供應商 200 家
  │   └── 團 50 個
  │
  ├── 🏘️ Local B（workspace_id: bbb）← 越南地接社
  │   ├── 員工 3 人
  │   ├── 供應商 80 家
  │   └── 案件 20 個（來自不同旅行社）
  │
  ├── 🏘️ Local C（workspace_id: ccc）← 日本地接社
  │   ├── ...
  │
  └── 🏛️ 旅行社 D（workspace_id: ddd）← 其他旅行社
      ├── ...

族譜法則：
  1. 每個租戶有自己的 workspace_id
  2. 資料完全隔離（RLS）
  3. 跨租戶溝通只透過「委託」機制
  4. 同一套系統、同一個核心表結構

附屬國加入帝國的流程：
  旅行社發委託 → 附屬國收到 → 自動建案
  → fork 世界樹枝條（核心表複製，不同 workspace_id）
  → 附屬國用同一套系統操作
  → 改動可追蹤（diff）

待建設：
  - [ ] 租戶註冊 + 審核
  - [ ] workspace 建立自動化
  - [ ] RLS 策略完善
  - [ ] 跨租戶委託傳遞機制
  - [ ] 租戶管理後台
```
