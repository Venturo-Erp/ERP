## 🔍 經驗教訓

### 1. SQL 查詢最佳實踐

**避免**: `workspaces!inner(code)` 語法  
**建議**: 分兩步查詢 (更清晰、更容易 debug)

### 2. 大小寫一致性

**原則**: 程式碼和資料庫的值要完全一致  
**檢查方式**: 直接查詢資料庫確認實際值

### 3. 權限檢查模式

**統一模式**:

1. 檢查登入狀態
2. 查詢用戶資料 (分兩步查詢)
3. 查詢 workspace 資料
4. 檢查權限

### 4. API 設計原則

**統一 API**: 一個 API 統一處理所有邏輯，避免邏輯分散  
**失敗 Rollback**: 中間失敗時自動清理已建立的資料  
**詳細日誌**: 記錄每個步驟，方便 debug
